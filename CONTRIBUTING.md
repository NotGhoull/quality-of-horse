# Contributing to Quality of Horse

## Getting started
### 1. Fork the repository and clone it  
To contribute, please fork this repository. We ask that you work in your own branch and then submit a Merge request (MR) to our main branch.

Note: We use fast-forward merging without merge commits, so please make sure to rebase your branch before submitting an MR.

## Setting up the project
### 2. Clone the repo
Clone the repository to your local machine
```bash
git clone https://gitlab.com/NotGhoull/quality-of-horse.git
cd quality-of-horse
```
### 3. Install dependencies
Ensure you have **Java 21** installed. We recommend using [Intellij IDEA](https://www.jetbrains.com/idea/) as your IDE, but you can use any Java IDE that supports Gradle.

## Coding standards
### 4. Code style guidelines
Currently, we do not have any strict guidelines, but we encourage you to follow general Java conventions and to be consistent with the existing codebase. We also recommend using descriptive variable and method names, and to include comments where necessary to explain complex logic.

### 5. Making a Merge request
When submitting a Merge request, please make sure to:
 - Label your MR appropriately
 - Have a descriptive title
 - Include a detailed description of the changes you made and why

### 6. Reporting issues
If you encounter any bugs or issues, please report them in the [issue tracker](https://gitlab.com/NotGhoull/quality-of-horse/-/issues). When reporting an issue, please include as much detail as possible, including steps to reproduce the issue, mod version, minecraft version, mod loader version, expected behavior, and any relevant logs or screenshots.

## Thank you!
Thank you for considering to contribute to Quality of Horse! We appreciate any and all contributions, whether it's code, documentation, bug reports, or just spreading the word about the mod. If you have any questions or need help getting started, feel free to reach out in our [Discord server](https://discord.lieonlion.dev).
